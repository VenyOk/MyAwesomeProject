# Проверка русскоязычного retrieval

`data/eval/russian_retrieval.json` — небольшой синтетический набор для
повторяемой проверки поиска по русскоязычным заметкам. В нём нет пользовательских
данных: это smoke-тест перед сменой embedding-модели, а не общий benchmark.

Скрипт измеряет:

- `Recall@k`: долю запросов, для которых хотя бы один релевантный документ попал
  в первые `k` результатов;
- `MRR@max(k)`: среднюю обратную позицию первого релевантного результата;
- имя модели, фактическую размерность векторов и размер набора.

По умолчанию скрипт работает только с моделями, уже лежащими в локальном кэше
Hugging Face. Он не скачивает веса и поэтому безопасен для CI и офлайн-проверки.

```powershell
python scripts/evaluate_embeddings.py `
  --model sentence-transformers/all-MiniLM-L6-v2 `
  --model intfloat/multilingual-e5-small
```

Если нужной модели ещё нет в кэше, один раз разрешите загрузку явно, затем снова
запускайте сравнение без этого флага:

```powershell
python scripts/evaluate_embeddings.py `
  --model intfloat/multilingual-e5-small `
  --allow-download
```

Для машинно-читаемого отчёта используйте `--json`; набор метрик можно изменить
через `--k 1 3 5`. Модели семейства E5 автоматически получают префиксы
`query:` для запросов и `passage:` для документов — так же, как retrieval в
приложении.

При выборе между моделями сначала сравнивайте `Recall@1`, `Recall@3` и `MRR@5`,
затем учитывайте размерность вектора, память и скорость на целевом компьютере.
Перед production-переключением дополните этот набор анонимизированными реальными
сценариями (имена, даты, номера задач, перефразировки) и повторите измерение.
